## Who I am

I'm a **vibe-coder** — I think in products, not in syntax.

My stack: clear problem definition + [Claude Code](https://claude.ai/code) doing the heavy lifting. I focus on *what* needs to exist and *why*. Claude Code handles the *how*. The result: working software, shipped fast, no tutorial hell.

> *"Don't learn to code. Learn to build."*
